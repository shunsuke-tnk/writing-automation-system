# ライティング自動化システム

このプロジェクトは、Cursorを活用して「自分らしい文章」を再現し、noteへの投稿を支援するためのシステムです。

## 使い方

### 記事生成プロンプト

すべての`.cursor/rules/`内のルールを厳格に遵守し、以下の条件でnote記事のドラフトを作成してください。

## 条件
- **テーマ**: (記事のテーマを簡潔に)
- **読者ターゲット**: (想定する読者層)
- **記事のゴール**: (読者がこの記事を読んで何を得られるか)
- **参考情報**: @knowledge/inputs/(日付)_(テーマ名).md
- **参考文体**: @knowledge/style-samples/ (このディレクトリ内の全記事)

## 出力
- 生成された記事は、`articles/(日付)_(テーマ名)/draft.md` というパスに保存してください。

## プロジェクト構造

```
writing-automation/
├── .cursor/                     # Cursor設定ディレクトリ
│   └── rules/                   # ルール定義
│       ├── 10_writing-style/    # ① 文体・スタイル設定
│       ├── 20_platform-rules/   # ② プラットフォーム特化ルール
│       └── 30_content-quality/  # ③ コンテンツ品質基準
├── knowledge/                   # ナレッジベース (AIの記憶)
│   ├── style-samples/           # 過去の良質な記事
│   ├── inputs/                  # インプット情報
│   └── outputs/                 # 過去の生成物
└── articles/                    # 生成された記事の保存場所
```

詳細な使い方は `MANUAL.md` を参照してください。

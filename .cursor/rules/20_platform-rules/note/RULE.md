# note特化フォーマット：Mobile-First Design

## 1. 構造と見出し

- **見出しレベル**: 本文の最上位見出しは `##` (h2) とし、小見出しは `###` (h3) とする。

- **セクション区切り**: 大きな話題の転換点には必ず `---` (水平線) を入れ、視覚的な休憩ポイントを作る。

## 2. スマホ最適化のリズム（重要）

- **改行と空白**: 
  - 2〜3文ごとに1行空行を入れる。
  - スマホの1画面が文字で埋まらないよう、適度に「余白」をデザインする。

- **強調のルール**: 
  - `**太字**` は1段落につき1箇所程度に絞り、本当に重要な部分だけが目に飛び込んでくるようにする。
  - 箇条書きは多用せず、物語の流れを優先するが、手順やリストを示す場合はnote標準の `*` を使用する。

## 3. 視覚要素

- **画像配置**: 
  - `##` 見出しの直後、または「情景描写」のセクションにAI生成画像を配置する指定を行う。
  - 代替テキスト (`alt`) には、画像の内容だけでなく、その画像が喚起する「感情」も一言添える。

- **引用と装飾**: 
  - 海外文献の直接引用には `>` を使用し、権威性を視覚的に区別する。
  - noteにはテーブル（表）機能が弱いため、データを示す場合はテーブルではなく、あえて箇条書きや画像化を検討する。

## 4. note固有の機能活用

- **リンクの扱い**: 記事内で参考URLを出す際は、単なるテキストリンクではなく、カード形式での埋め込みを想定した配置にする。

- **目次の考慮**: `##` 見出しが自動的に目次になることを意識し、見出しだけで記事のベネフィットが伝わるようにする。

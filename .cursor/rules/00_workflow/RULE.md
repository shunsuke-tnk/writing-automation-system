# 執筆ワークフロールール

## 1. 記事執筆時の基本フロー

記事を執筆する際は、必ず以下の順序で作業を行う：

1. **執筆**: `.cursor/rules/` 内のすべてのルールに従って記事を執筆する
2. **フォルダ作成**: `knowledge/outputs/` 内に `yyyy-mm-dd_タイトル` 形式のフォルダを作成する
3. **下書き保存**: 執筆した原稿を `draft.md` というファイル名で保存する
4. **完成版保存**: 修正・完成した原稿は同じフォルダ内に `article.md` として新規保存する

## 2. フォルダ命名規則

- 形式: `yyyy-mm-dd_タイトル`
- 日付: 執筆日（例: 2025-12-23）
- タイトル: 記事を識別できる短いタイトル（日本語可、記号は避ける）
- 例: `2025-12-23_お金が集まる人の脳内`

## 3. ファイル構造

```
knowledge/outputs/yyyy-mm-dd_タイトル/
├── draft.md    # AIが生成した下書き（最初に作成）
└── article.md  # 完成した記事（修正後に作成）
```

## 4. 自動実行ルール

ユーザーから記事執筆の指示があった場合、明示的な指示がなくても以下を自動で実行する：

- 記事執筆完了後、自動的にフォルダを作成する
- `draft.md` として保存する
- フォルダ名は記事タイトルから適切に命名する
